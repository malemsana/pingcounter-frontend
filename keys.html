The Modern Metric Engine.

PingCounter is a high-performance, developer-first service designed for real-time tracking, visual analytics, and secure metric aggregation. Whether you're tracking page loads, API hits, or user interactions, we provide the infrastructure to capture and visualize it instantly.

High Speed

Redis-backed ingestion capable of handling thousands of pings per second with minimal latency.

Secure Scopes

Granular access control ensuring your data is only visible to authorized keys and users.

Visual-First

Beautiful SVG badges built-in, with zero-config embedding for any web environment.

API Scopes & Security

API keys are scoped to specific operations. This allows you to expose keys for different parts of your application without risking the security of your full dataset.

ping

Write-Only Access

Allows triggering event counts. This is the safest key to expose in client-side code. It cannot be used to retrieve statistics or view badges.

read

Data Retrieval Access

Allows viewing event stats and generating authenticated badges. Use this key in backend environments to pull metrics securely.

admin

Full Control

Grants both ping and read permissions. This is the recommended scope for administrative internal tools.

Advanced Security Rules

Allowed Origins (CORS)

Restrict `ping` requests to specific domains. If set, the server will reject any request coming from an origin not in your whitelist.

Allowed IPs

Hard-lock your data retrieval. Restrict `read` and `admin` operations to specific static IP addresses (e.g., your office or server IP).

Ingestion Engine

To increment a counter, send a request to the ingestion endpoint. We support both GET and POST to make integration as simple as possible.

GET POST /p/:api_key/:project/:event
Parameters
api_key A key with ping scope
project The project namespace slug
event The specific event identifier
Integration Methods
JavaScript Fetch (Modern Web)
// Trigger on button click
fetch('https://pingcounter.onrender.com/p/pc_123/site/signup', { method: 'POST', mode: 'no-cors' // Passive tracking });
Image Pixel (Zero JS)
<!-- Tracks on page load -->
<img src="https://pingcounter.onrender.com/p/pc_123/site/page-view" width="1" height="1" style="display:none" />

How does Image Tracking work?

When a browser encounters an img tag, it automatically sends a GET request to the URL to retrieve the image. PingCounter intercepts this request, increments your event count, and returns a 204 No Content response.

  • Works even if the user has JavaScript disabled.
  • Zero performance impact on the main thread.
  • Perfect for Email tracking, Markdown files (GitHub READMEs), and static sites.
Advanced: Analytics Beacon

For tracking data when a user leaves a page without delaying the navigation:

window.addEventListener('visibilitychange', () => { if (document.visibilityState === 'hidden') { navigator.sendBeacon('https://pingcounter.onrender.com/p/pc_123/site/exit'); } });

Visual Badges

Visualize your metrics anywhere with zero dependencies. Badges are rendered dynamically as SVGs on our edge servers.

Public Badge

Requires the event to have "Public Access" enabled. Uses a secure hash slug.

/badge/:public_slug?style=flat

Authenticated Badge

Works for private events. Requires a key with read scope passed in the URL.

/api/v1/badge/:key/:project/:event
Supported Styles
flat flat-square plastic for-the-badge social minimal glass

Developer Workflows

Zero-Config Ingestion

You don't need to create projects or events manually in the dashboard. If a valid API key triggers an ingestion call for a project/event that doesn't exist, we'll create it instantly in pending mode.

Auto-Approve Events

For highly dynamic environments, enable **Auto-Approve** in Project Settings. New events will bypass the pending queue and start tracking instantly the moment they are first triggered.

The Pending Queue

By default, unknown events are put in the "Pending" state. This protects your metric history from accidental spam or "ghost" events created by typos in your integration code.
